# Market Simulator Benchmark

This package adds a benchmark-oriented market simulator track on top of the existing repository.

Scope:

- immediate-clearing surrogate baseline
- fixed-delay speed-bump baseline
- frequent batch auction scenarios
- adaptive-window heuristic baselines, including balanced, order-flow, and queue-load variants
- adapter-driven policy baselines (`Policy-BurstAware-100-250ms`, `Policy-LearnedLinUCB-100-250ms`, `Policy-LearnedTinyMLP-100-250ms`, `Policy-LearnedOfflineContextual-100-250ms`, `Policy-LearnedFittedQ-100-250ms`, `Policy-LearnedOnlineDQN-100-250ms`)
- stronger online-learning artifacts via a prioritized Double-DQN style controller
- agent-based order flow
- a strategic-agent robustness layer with inventory-aware makers, trend-reactive retail flow, signal-scaled informed traders, and dislocation-sensitive arbitrageurs
- ledger-aware settlement checks
- fairness, welfare, and market-quality metrics
- step-wise `Reset/Step/Observe/Metrics` API for downstream control loops
- minimal gym-style adapter via `NewAdapter`, with batch-window, risk-scale, tie-break, release-cadence, and price-aggression controls plus reward-bearing timesteps
- schema documentation in `docs/neurips_track/ENVIRONMENT_SCHEMA.md`
- logged-trajectory training and held-out regime evaluation artifacts
- fitted-Q learning-curve artifacts over held-out regimes
- mechanism ablations and agent/workload ablations
- parameter-grid sweeps over arbitrage intensity and maker quote width
- parameter-cube sweeps over retail intensity, informed intensity, and maker quote width
- unified parameter-hypercube sweeps over arbitrage intensity, retail intensity, informed intensity, and maker quote width
- compact hypercube summary artifacts for main effects, high-low contrasts, and retail-conditioned arbitrage deltas
- fitted hypercube response-surface artifacts for compact statistical analysis
- multiseed controller Pareto frontier artifacts

The current learned-controller stack now includes:

- `Policy-LearnedLinUCB-100-250ms`: contextual linear bandit over the discrete adapter action bundle
- `Policy-LearnedTinyMLP-100-250ms`: burst-aware supervised warm-start plus gradient-based policy updates
- `Policy-LearnedOfflineContextual-100-250ms`: offline contextual value baseline trained from logged trajectories generated by burst-aware, LinUCB, TinyMLP, and random behavior policies
- `Policy-LearnedFittedQ-100-250ms`: offline fitted-Q style controller trained on logged trajectories from burst-aware, LinUCB, TinyMLP, offline contextual, and random behavior policies, then evaluated on held-out regimes
- `Policy-LearnedOnlineDQN-100-250ms`: replay-based online DQN-style controller trained directly through the adapter action space, with held-out learning-curve artifacts
- `Policy-LearnedDoubleDQN-100-250ms`: prioritized replay plus Double-DQN target variant reported through a held-out learning-curve artifact rather than the main multiseed baseline table

The paper-facing welfare decomposition is intentionally narrow. The benchmark centers on:

- `retail_surplus_per_unit`
- `retail_adverse_selection_rate`
- `surplus_transfer_gap`

Secondary diagnostics such as `arbitrageur_surplus_per_unit` and `welfare_dispersion` remain in artifacts and rewards, but they are no longer the core manuscript claim.

Artifact boundary:

- `binance_spot_*_facts.*` and `binance_spot_*_provenance.*` are real-data artifacts derived from public Binance Spot downloads.
- `simulator_*` artifacts are synthetic benchmark outputs.
- `simulator_calibration_target_table.*` and `simulator_calibrated_vs_market.*` are mixed artifacts comparing real-data envelopes against simulator outputs.
