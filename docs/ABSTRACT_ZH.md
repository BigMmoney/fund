# 中文摘要

本文面向市场基础设施系统，设计并实现了一个以账本优先结算为核心的频繁批量竞价交易原型。系统将撮合引擎、双分录账本、风险控制模块与事件驱动状态传播机制解耦为独立组件，目标不是单纯追求低延迟，而是在重放、部分成交与策略切换场景下保持确定性与可验证性。具体而言，撮合层采用 Frequent Batch Auction（FBA）机制，在固定批窗口内聚合订单并进行统一价格清算；结算层采用显式双分录账本模型，通过 `op_id` 去重、账户余额校验与提交日志保证状态变更的原子性与可追踪性；风险层通过市场状态机与分级 kill switch 将交易阻断、提现阻断、链上签名阻断和只读模式编码为可执行策略。

本文基于仓库中的合成工作负载对系统进行评估。在单市场、单 outcome、200 对买卖订单的场景下，100 ms 批窗口下系统可达到 4958.8 orders/s，500 ms 批窗口下为 832.8 orders/s，1000 ms 批窗口下为 407.4 orders/s；对应 p50 延迟分别为 80.66 ms、480.29 ms 和 981.85 ms。实验结果表明，较短批窗口具有更好的响应性和吞吐表现，而较长批窗口则以更高的等待时间换取更强的时间聚合特性，体现了公平性与延迟之间的典型工程权衡。

除性能评估外，本文将系统正确性主张落实为可执行不变量测试，包括：清算价选择的确定性、买卖两侧成交量守恒、账本资金守恒、重复 `op_id` 重放的幂等性，以及风险状态与 kill switch 限制的单调性。与仅展示产品功能的实现不同，本工作强调“可复现的系统行为”与“可验证的结算语义”。整体来看，该原型虽然尚未具备生产级持久化、分布式事件总线和多节点容错能力，但已经形成一套面向研究和工程演示的市场系统 artifact，可作为后续开展持久化恢复、对抗性订单流仿真、形式化状态机验证以及更大规模实验评估的基础。

**关键词：** 市场基础设施，频繁批量竞价，双分录账本，确定性撮合，重放安全，风险控制
