# 中文摘要

本文围绕一个研究问题展开：以账本优先结算为核心的频繁批量竞价（Frequent Batch Auction, FBA）市场架构，能否在保持短批窗口下可用吞吐能力的同时，提供确定性的结算语义与可执行的正确性保证？为回答该问题，本文设计并实现了一个模块化市场基础设施原型，将撮合引擎、双分录账本、风险控制模块与事件驱动状态传播层解耦为独立组件。系统的核心目标不是单纯追求最低延迟，而是在重放、部分成交和控制策略切换等场景下保持可验证、可复现的系统行为。

在方法上，撮合层采用 FBA 机制，在固定批窗口内聚合订单并执行统一价格清算；当多个价格点产生相同成交量时，系统采用显式且确定性的最低价 tie-break 规则；在成交分配阶段，系统采用按比例分配加最大余数法（largest remainder），保证买卖两侧成交量严格守恒。结算层采用显式双分录账本模型，通过 `op_id` 去重、账户余额校验与提交日志记录，保证状态变更的原子性、可追踪性和重放安全。风险层则通过市场状态机和分级 kill switch，将阻断新交易、阻断提现、阻断链上签名与只读模式编码为可执行的控制语义。

本文使用仓库内的合成工作负载进行评估，并引入 immediate-clearing surrogate 作为对照基线。在单市场、单 outcome、200 对买卖订单的场景下，immediate surrogate 可达到 144,744.0 orders/s，p50 延迟为 1.71 ms；FBA-100ms 为 4,978.0 orders/s，p50 延迟为 80.35 ms；FBA-250ms 为 1,736.2 orders/s，p50 延迟为 230.39 ms；FBA-500ms 为 833.4 orders/s，p50 延迟为 479.96 ms。结果表明，随着批窗口增大，系统吞吐下降、等待延迟上升，这一代价是引入显式批处理语义所必须支付的工程成本。本文不将该结果误写为“已实证证明公平性提升”，而是将其严格限定为对 batching 成本与系统响应性权衡的定量描述。

除性能评估外，本文还将系统正确性主张形式化为可执行不变量，包括：成交量守恒、非系统账户余额非负、账本资金守恒、重复 `op_id` 重放的幂等性，以及风险限制在更高 kill switch 等级下保持单调增强。与一般工程项目文档不同，本文强调“研究问题 + 算法 + 不变量 + 实验”的完整结构，使该原型从系统说明提升为可用于研究展示与后续投稿扩展的市场基础设施 artifact。尽管该实现尚未具备生产级持久化、分布式事件总线与多节点容错能力，但它已经为后续开展持续撮合基线对照、随机到达订单流实验、形式化公平性指标以及恢复路径验证提供了清晰基础。

**关键词：** 市场基础设施，频繁批量竞价，双分录账本，确定性撮合，重放安全，风险控制
